Flexible Functions

At the time this documentation was written, NaNMath's functions were not designed for Intervals from IntervalArithmetic or Nums from Symbolics.

OceanSonar.Flex is designed to wrap functions of the same names in NaNMath and dispatch to the respectively appropriate "backend" - quoted because Nums and Intervals get passed to Base. "Normal" numbers passed to functions in NaNMath and OceanSonar.Flex will output NaN for invalid inputs e.g. sin(-1::Real) or sqrt(-1::Real), whereas the equivalent functions in Base throw an error.

OceanSonar.Flex — Module
OceanSonar.Flex :: Module

A module containing dispatching methods imported from other modules.

Specifically:

  • Contains mathematical functions that return NaN instead of erroring on mathematically invalid inputs,

such as sqrt(-1::Number).

  • derivative which dispatches appropriately

to DataInterpolations.derivative, ForwardDiff.derivative (not yet implemented), or Symbolics.derivative (not yet implemented).

The OceanSonar.Flexible functions are a temporary implementation until NaNMath functions are defined to fallback to their respective Base functions.
